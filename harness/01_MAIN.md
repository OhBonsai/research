回到我所在的业务是云电脑，在过去几年的发展，积累了大量的 B 端客户，非常多都是用 windows 云电脑进行安全办公的。对于其他云产品或者行业来说，想办法让客户用， 所以我们推出了一个产品。 但是遇到困惑





当我在使用ClaudeCode/OpenCode 生成 PPT 的时候，我发现相较于 直接让 LLM 去生成 PPT，换成另外一种思路，先让 LLM 生成 网页版 PPT，然后让 LLM 将 网页版转化为 PPT。 这样出来的效果贼好。除此之外 OpenClaw 火爆下带来的安全问题，我接触下来大多数看过 OpenClaw 源码或者懂它在做什么的工程师都不想用，觉得太危险了，就像把给一个偶尔中二的小孩开了免密无限额支付 。

这背后引发我的一个思考。当我再说让Agent更有用的时候，在不能够改变Model本身时，我们能做的事情有哪些？这些事情的集合在工程上领域范畴是什么？ 具体而言有哪些可以调整和选择系统元素，又有哪些设计原则，和系统约束？

这些困惑在我看到Harness的时候，通了！ 就像汽车不能只能有发动机，汽车要有刹车系统，要有座舱，要有悬挂。 一个 AI Agent 不能只有模型，也要有操作模型的工程方法。 也就是Harness Engineering（模型操作系统）

# Mini Spec：AI Agent 评测 Case 定义

## Case 列表

### MINI-01（SHORT）
- **场景：** 新员工入职文档核对
- **测试目标：** Progressive Disclosure（渐进展开）
- **轮次目标：** 7 轮
- **种子数据需求：**
  - 员工信息表（5 人，CSV）
  - 入职检查清单（12 项，YAML）
  - 公司制度摘要（Markdown）
- **用户指令序列：**
  1. "这是新入职的 5 名员工资料和入职检查清单，请帮我核对每人的材料完整度"
  2. "张明的社保材料缺了哪些？"
  3. "帮我生成一份催办邮件给张明，列出缺失项"
  4. "李芳的学历认证通过了吗？"
  5. "汇总所有人的核对状态，生成进度表"
  6. "王刚的紧急联系人信息对吗？和他的档案交叉校验"
  7. "生成最终的入职核对报告"
- **Probe 问题：**
  - [recall] "张明缺了哪些社保材料？"
  - [artifact] "生成了哪些文档？内容与源数据一致吗？"
  - [plan] "如果有人的材料 3 天内补不齐，下一步怎么处理？"
- **成功标准：**
  - 正确识别每人缺失项
  - 催办邮件包含具体缺失列表
  - 最终报告数据与源文件一致

### MINI-02（MID）
- **场景：** 季度预算审批流程
- **测试目标：** Context Compaction（上下文压缩）+ Structured Assembly（结构化装配）
- **轮次目标：** 25 轮
- **种子数据需求：**
  - 4 个部门的预算申请表（YAML，含明细）
  - 历史预算数据（去年同期，CSV）
  - 审批规则文档（Markdown，含阈值）
  - 高管批示记录（3 条，YAML）
- **用户指令序列：**
  1. "这是本季度 4 个部门的预算申请，请先做个总览分析"
  2. "研发部的云服务费用比去年增长了多少？合理吗？"
  3. "按审批规则，哪些项目需要 VP 审批？"
  4. "（继续逐部门审核，约 15 轮类似操作）"
  5. "综合所有审批意见，生成最终预算批复文件"
  6. "如果总预算要砍 15%，建议从哪里削减？"
- **Probe 问题：**
  - [recall] "研发部云服务费用去年和今年分别是多少？"
  - [decision] "为什么建议削减市场部的差旅预算而不是研发部的设备预算？"
  - [artifact] "最终预算批复文件中，各部门批复金额与审批过程中的结论一致吗？"
  - [plan] "如果 CEO 要求追加一个新项目预算，应该如何调整现有分配？"
- **成功标准：**
  - 每个部门的预算增减分析有数据支撑
  - 审批规则正确应用
  - 最终文件与过程数据一致

### MINI-03（LONG）
- **场景：** 跨部门系统迁移项目管理
- **测试目标：** Death Spiral Detection（死亡螺旋检测）+ Tool Output Backpressure（工具输出背压）
- **轮次目标：** 80 轮
- **种子数据需求：**
  - 项目 WBS（60 个任务，含依赖关系，YAML）
  - 团队花名册（20 人，CSV）
  - 里程碑计划（8 个节点，YAML）
  - 风险登记册（15 条，YAML）
  - 每周进度报告模板（Markdown）
- **用户指令序列：**
  1. "这是系统迁移项目的完整计划，请先分析关键路径"
  2. "第一阶段的 3 个任务分配给谁最合适？考虑技能和可用性"
  3. "（每周模拟进度汇报 + 问题处理，约 60 轮）"
  4. "数据库迁移延期了 2 周，重新评估整体排期影响"
  5. "项目经理离职了，如何调整分工？"
  6. "生成面向管理层的项目状态总结"
- **Probe 问题：**
  - [recall] "WBS 中任务 T-15 的前置依赖是什么？"
  - [decision] "为什么选择周末做数据库切换而不是工作日？"
  - [plan] "如果再延期 1 周，哪些里程碑受影响？"
  - [artifact] "进度报告中的完成率与任务状态是否一致？"
- **成功标准：**
  - 关键路径计算正确
  - 资源分配考虑技能匹配和可用性
  - 延期影响分析覆盖所有依赖任务
  - 死亡螺旋场景中 Agent 主动示警
